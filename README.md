# 새록이 (dementia-care)

> 어르신을 위한 하루 3분 치매예방 웹앱 — 그리고 **개인 리눅스 데스크탑을 Docker로 서버화해 안전하게 배포**하는 학습 프로젝트

"기억이 새록새록"이라는 뜻의 **새록이**는 인지 정상 어르신이 매일 3분, 5단계로 두뇌를 쓰는 웹앱입니다. 다만 이 저장소의 진짜 목적은 앱 그 자체보다 **개인 컴퓨터를 컨테이너로 감싸 인터넷에 안전하게 노출하는 전 과정**을 직접 설계하고 검증하는 데 있습니다.

---

## 무엇을 만들었나

- 로그인 없이 **익명 UUID**로만 사용자를 구분 (개인정보 미수집)
- 하루 1회, **5단계 세션** (기분 → 준비운동 → 본운동 → 회상 → 미션)
- **정답 판정·난이도 조정을 서버에서만** 수행 (프런트는 정답을 모름)
- 연속 참여를 **도장판**으로 표시 (점수·정답률은 노출하지 않음 — 실패감 방지)
- 큰 글자·큰 버튼·넓은 간격 등 **어르신 UI 규격**을 토큰으로 관리

## 아키텍처

```
[사용자]
   │  HTTPS
   ▼
[Cloudflare]  ── Access 인증(지정 사용자만) ──┐
   │  Tunnel (아웃바운드 전용)                 │
   ▼                                          │
┌─────────────── Docker (전용 네트워크) ───────────────┐
│  cloudflared ──▶ web (Next.js) ──▶ api (FastAPI)     │
│                                        │             │
│                                   SQLite (볼륨)       │
└──────────────────────────────────────────────────────┘
     ▲ 호스트에 열린 포트 없음 (ports 매핑 0개)
```

- **web** — Next.js. 화면을 그리고, `/api/py/*` 요청을 api로 중계(rewrite)
- **api** — FastAPI. 세션·판정·난이도·기록을 담당
- **SQLite** — 사용자·완료기록·진행상태를 영구 저장 (named volume)
- **cloudflared** — Cloudflare Tunnel 커넥터. 바깥으로 나가는 연결만 사용

## 실행 방법

> 로컬 개발과 컨테이너 실행이 분리되어 있습니다. 비밀값은 저장소에 포함되지 않으며, 각자 환경에서 설정해야 합니다.

### 사전 준비

- Docker / Docker Compose
- Cloudflare 계정 및 관리 중인 도메인 (터널·Access용)
- 터널 토큰 등 비밀값은 `.env`에 직접 설정 (저장소에 미포함, `.gitignore` 처리됨)

### 컨테이너 실행

```bash
# api·web 빌드
sudo docker compose build api web

# 터널 포함 전체 기동 (.env의 TUNNEL_TOKEN 필요)
sudo docker compose --profile tunnel up -d

# 상태 확인
sudo docker compose ps

# 정리 (데이터 볼륨은 유지)
sudo docker compose down
```

> 실행 환경에 따라 `sudo` 여부가 다를 수 있습니다.

## 보안 설계

개인 기기를 인터넷에 노출하는 프로젝트인 만큼, 배포 전 체크리스트를 통과한 뒤에만 터널을 열도록 설계했습니다.

- **호스트 포트 매핑 0개** — 컨테이너는 전용 네트워크 안에서만 통신, 외부 노출은 터널 하나로 한정
- **Docker 소켓 미마운트 · privileged 미사용**
- **비-root 실행 (1000:1000) · `cap_drop: ALL` · `no-new-privileges` · `read_only`**
- **비밀값은 `.env`로 분리**하여 이미지·저장소에 포함하지 않음
- **Cloudflare Access**로 지정한 사용자만 접근 허용
- **정답·난이도 로직은 서버에만** 존재 (프런트에 정답을 내려보내지 않음)

자세한 위협 모델과 판단 근거는 `docs/`를 참고하세요.

## 배운 것

이 프로젝트는 코드를 빨리 짜는 것보다 **개념을 이해하고 검증하는 것**에 무게를 두었습니다.

- 문서를 먼저 쓰고(PRD·위협모델·API명세), 코드는 그 계약을 따르게 했습니다.
- 각 단계마다 "됐겠지"가 아니라 **실제로 확인**했습니다 — 커밋 전 실측, 재시작 후에도 데이터가 남는지, 호스트에 포트가 안 열렸는지, 인증이 실제로 막는지.
- 임시로 처리한 것은 결정 기록에 남겨두었다가 **반드시 되돌아와 해소**했습니다.

설계 판단의 근거와 임시조치·해소 이력은 `docs/decisions.md`에 단계별로 남겼습니다.
