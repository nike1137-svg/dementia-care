## 1. 개요

### 1.1 이 프로젝트의 진짜 목적 ★

> **리눅스 데스크탑(marcus-desktop)을 Docker로 서버화해 외부에 서비스해보는 것.**

치매예방 앱은 **그 위에 올릴 소재**다. 앱이 목적이 아니다.

이 문장이 모든 판단의 기준이 된다:

| 판단 | 근거 |
|---|---|
| 클라우드 배포 안 함 | 내 컴퓨터를 서버로 쓰는 게 목적이므로 |
| 보안에 민감하게 반응 | 개인 실기기를 인터넷에 붙이는 일이므로 |
| 가용성은 신경 안 씀 | 평가자도 사용자도 없음. 꺼져도 무방 |
| 복잡한 구성 지양 | Docker를 처음 쓰므로 변수를 줄인다 |

### 1.2 성격

- **KDT 과제 제출물이 아니다.** 강의 내용을 응용한 **개인 1차 프로젝트**
- 강의의 보안 체크리스트는 **남에게 보일 기준이 아니라 스스로에게 적용할 기준**
- GitHub는 **코드 보관·README 용도**이지 배포처가 아니다

### 1.3 앱의 한 줄 정의

> 진단하지 않습니다. 매일 3분, 뇌를 쓰는 습관을 만듭니다.

### 1.4 목표

- 어르신이 **하루 3분 세션을 혼자 완주**할 수 있다
- **새로고침·다른 기기에서도 기록이 남는다** ← 백엔드가 실제로 도는 증거
- 도커 컨테이너 3개가 **격리된 채로** 돌고, 터널로만 외부에 열린다

### 1.5 비목표 (v1에서 명시적으로 제외)

| 제외 | 이유 |
|---|---|
| 치매 진단·선별 | 의료기기 규제 경계선 |
| MCI·치매 진단자 대상 | 의료적 관리 영역 |
| LLM 기반 자유 대화 | 과금 발생 + 범위 초과 |
| 음성 입력(STT) | 과금 가능성 |
| 회원가입·로그인 | 어르신 진입장벽 1순위 |
| 모바일 네이티브 앱 | 웹으로 충분 |
| 고가용성·무중단 | **목적에 없음** |

## 3. 핵심 기능 *(v1에서 변경 없음)*

### 3.1 세션 구조 — 모든 세션 동일 골격

| 단계 | 시간 | 내용 | 인지 목적 |
|---|---|---|---|
| ① 인사·기분 | 20초 | 오늘 기분 3택 | 정서 체크·진입 |
| ② 워밍업 | 30초 | 오늘 날짜·요일 | 지남력 |
| ③ 오늘의 활동 | 90초 | 주 영역 과제 1개 | 핵심 인지 자극 |
| ④ 회상 한 조각 | 40초 | 회상 유도 질문 | 장기기억·정서 |
| ⑤ 마무리 미션 | 20초 | 생활 실천 1개 | 신체·생활 연결 |

### 3.2 12주 커리큘럼

| 주차 | 인지 영역 | 대표 활동 |
|---|---|---|
| 1~2 | 지남력 | 날짜·계절·명절, 동네 지도 |
| 3~4 | 주의·집중 | 숫자 거꾸로 세기, 표적 반응 |
| 5~6 | 기억력 | 단어 3개 지연 회상 |
| 7~8 | 언어 | 끝말잇기, 범주 유창성, 속담 완성 |
| 9~10 | 계산·실행 | 장보기 계산, 순서 정하기 |
| 11~12 | 시공간·회상 | 사진 설명, 옛날 물건 이야기 |

- 주 5일(월~금), 주말 쉼 · 6영역 × 2주 = 1사이클 · 13주차부터 난이도 올려 재순환

### 3.3 난이도 자동 조정

| 단계 | 유형 | 예시 |
|---|---|---|
| 1 (하) | 2택, 답이 뻔한 보기 | "일주일은 며칠이에요? (7일 / 10일)" |
| 2 (중) | 4택 + 단서 | "월요일 다음은 무슨 요일이에요? (화·수·목·금)" |
| 3 (상) | 4택, 단서 없음, 헷갈리는 보기 | "정월대보름에 하는 건? (부럼 깨기·송편 빚기·세배하기·팥죽 먹기)" |

> ★ 어르신은 타이핑을 못 한다(§2.1). **모든 문항은 탭으로만 답한다.** 주관식(타이핑 입력)은 배제한다.
> ★ **보기 순서는 서버가 섞는다** (세션+문항 기준 결정적 셔플). 위치 학습은 막고, 세션 중에는 순서가 고정된다.

- 3회 연속 성공 → 상승 / 2회 연속 실패 → 하강
- 별도 평가 화면 없음
- ★ **계산은 전부 서버에서** (위협모델 C-7)

### 3.4 오답 처리 원칙 ★ 최우선 UX 규칙

- **정답을 즉시 알려주지 않는다**
- 1차 오답: "비슷해요, 하나만 더 생각해볼까요?"
- 2차 오답: "괜찮아요, 오늘은 여기까지도 잘하셨어요" → 다음 단계
- **"틀렸습니다" / "오답입니다" 문구 절대 금지**
- 실패 경험 = 이탈 1순위 원인

## 4. 화면 정의

| 화면 | 경로 | 내용 |
|---|---|---|
| 홈 | `/` | 세션 시작 버튼 1개 + 연속 참여일 |
| 세션 | `/session` | ①~⑤ 진행 (한 화면 = 한 단계) |
| 완료 | `/done` | 오늘의 미션 + 격려 + 연속일 |
| 기록 | `/history` | 최근 7일 도장판 |

### 4.1 UI 요구사항 (어르신 전용)

- 기본 폰트 **20px 이상**, 문항 **24px 이상**
  - 크기는 **하한이며 상한이 아니다.** 사용자가 글씨를 키우면 그에 따라 커진다 (px 고정 금지, `max()` + `rem`)
- 글자 굵기 **500 이상** — 얇은 폰트는 노안에 흐려 보인다
- 터치 영역 **60×60px 이상**
- 버튼 사이 간격 **최소 24px** — 어르신은 손 떨림으로 인접 버튼을 오터치한다. 오터치가 오답으로 기록되면 난이도 자동조정이 인지가 아닌 손 떨림을 반영하게 된다
- 화면당 선택지 **최대 4개**
- 명도 대비 **7:1 (WCAG AAA)** — 화면당 요소가 적어 색을 절제하면 자동 달성된다
- **색만으로 정보를 전달하지 않는다** — 노화로 색 구분력이 떨어진다. 색 + 글자/모양을 함께 쓴다
- **스크롤 없이** 한 화면에 완결
- 애니메이션 최소화
- 뒤로가기 항상 노출
- 모든 호출에 **로딩 / 성공 / 에러** 3상태 화면 준비

## 6. 기술 스택 (확정)

| 레이어 | 선택 | 근거 |
|---|---|---|
| 프런트 | **Next.js** | 기존 경험 보유 |
| 백엔드 | **FastAPI (Python3)** | ★ 마커스님이 공부 중인 언어 — **협상 불가 제약** |
| DB | **SQLite** | 이 앱의 요구를 충족 + 컨테이너·실패지점 축소 |
| 컨테이너 | **Docker Compose** | 격리를 위해 **필요해서** 도입 (학습 목적 아님) |
| 외부 노출 | **Cloudflare Tunnel** (신규) | 포트 개방 불필요 |
| 접근 제어 | **Cloudflare Access** | 개발 기간 봇 차단 |
| 주소 | **care.dodami-ai.com** | 기존 도메인 서브도메인 (추가 비용 0) |
| 저장소 | **nike1137-svg/dementia-care** | 코드 보관·README |

### 6.1 왜 SQLite인가 (v1의 PostgreSQL에서 변경)

1. 이 앱의 요구를 완전히 충족 — PostgreSQL의 초과 능력은 쓸 데가 없다
2. 컨테이너가 줄면 공격 표면과 실패 지점이 같이 준다
3. **Docker를 처음 쓰므로 변수를 줄인다** (시작 순서·비밀번호·볼륨 권한 문제가 통째로 사라짐)

> 나중에 필요해지면 교체한다. 지금 미리 짊어질 이유가 없다.

### 6.2 왜 CORS가 안 생기는가

브라우저는 항상 `care.dodami-ai.com/api/py/...` 만 호출한다.
Next.js가 `next.config.js` 의 rewrites로 컨테이너 내부 `http://api:8000/...` 으로 중계한다.
브라우저 입장에선 **같은 출처(same-origin)** → CORS 자체가 성립하지 않는다.

> CORS는 Python 때문이 아니라 **별도 서버를 다른 주소에 두면** 생기는 것이었다.

## 7. 인프라 구성 (신설)

### 7.1 컨테이너 3개

| 컨테이너 | 역할 | 외부 노출 |
|---|---|---|
| `web` | Next.js — 화면 + API 프록시 | cloudflared 경유만 |
| `api` | FastAPI + SQLite 파일 | ❌ 없음 |
| `cloudflared` | 터널 | 아웃바운드만 |

- SQLite 파일은 `api` 에 **named volume** 으로 부착 (컨테이너 지워도 데이터 유지)

### 7.2 격리 요구사항 ★ 절대 준수

- 기존 6개 OpenClaw 에이전트(자비스/테크/혜영/지연/지수/오메가)에 **영향 0**
- 기존 n8n(5678) 건드리지 않음
- 온담이 관련 파일·설정 **일절 접근 금지**
- **전용 도커 네트워크.** 기존 네트워크에 attach 금지, 기본 bridge 사용 금지
- **`ports:` 매핑 0개** — 호스트 포트를 새로 열지 않는다
- **cloudflared는 호스트가 아니라 격리 네트워크 안 컨테이너로 실행**
  → 오설정해도 n8n·온담이에 **도달 경로가 없다**

> 상세 통제는 `위협모델 v1` §4 참조. 배포 전 §6 체크리스트 전항목 통과 필수.

### 7.3 💰 비용

| 항목 | 비용 | 카드 |
|---|---|---|
| Docker / Next.js / FastAPI / SQLite | 0원 (오픈소스) | 불필요 |
| Cloudflare Tunnel | 0원 (대역폭 무제한) | 불필요 |
| Cloudflare Access | 0원 (50명까지) | **불필요** |
| 서브도메인 | 0원 (도메인 기보유) | — |
| GitHub | 0원 | — |
| 서버 | 0원 (자체 하드웨어) | — |
| **합계** | **0원** | **카드 미등록 = 과금 경로 없음** |

⚠️ 미확인: 무료 플랜 데이터 전송 한도(3자 자료에 10GB 언급). 설정 시 공식 요금 페이지 확인.
⚠️ 가입 중 **카드 입력 화면이 뜨면 중단**하고 재검토.
⚠️ 향후 LLM·STT 연동 시 과금 발생 → v1 범위 밖.

### 10.1 문항 라이선스 주의
국민건강보험공단·중앙치매센터 인지훈련 자료집은 무료 공개이나 **"자유 활용"이 비영리 현장 기준일 가능성**이 있음. 애매하면 **문항은 자체 제작하고 구조·난이도 체계만 참고**할 것.
